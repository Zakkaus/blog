# Gentoo Article Synchronisation Guide

Working notes for porting and maintaining the four Gentoo articles across three languages:

- `2025-10-02-gentoo-m-series-mac/` — Asahi Linux on Apple Silicon (ARM64)
- `2025-11-25-gentoo-install-base/` — Base install (AMD64)
- `2025-11-25-gentoo-install-desktop/` — Desktop configuration
- `2025-11-25-gentoo-install-advanced/` — Advanced optimisation (LTO, kernel, RAID)

Each article exists in three locales: `index.en.md`, `index.zh-cn.md`, `index.zh-tw.md`. The same heading structure must hold in all three; only the things listed below should differ.

## Overview

| Locale | Source / role | Audience defaults |
| :--- | :--- | :--- |
| `zh-cn` (Simplified) | Original / source of truth for new content | Mainland China — BFSU/Tsinghua/USTC mirrors, `Asia/Shanghai`, Mainland app examples (`com.qq.QQ`, `com.tencent.WeChat`, `flclash-bin`), gentoo-zh overlay encouraged |
| `zh-tw` (Traditional) | Converted from `zh-cn` via OpenCC, then word-fixed | Taiwan — TWAREN mirror, `Asia/Taipei`, Taiwan-localised terminology (see "Wording" below) |
| `en` | Manually adapted from `zh-tw` for international readers | International — official Gentoo mirrors via `mirrorselect`, `UTC` as the timezone example, no region-specific apps, broader font set |

## Heading / structure rules

- Every `## n.`, `### n.x`, `#### n.x.y` heading must appear in all three locales with the same numbering. Numbering errors caught in past reviews: install-base section 7 was `7.1 → 7.3` instead of `7.1 → 7.2`; install-base previously had `### 11.1` and `### 11.2` without a `## 11.` parent.
- Heading anchor IDs are auto-generated by Hugo's Goldmark; keep the visible text the same shape across languages so anchors line up (e.g. `## 6. Profile, System Settings & Localization {#step-6-system}` keeps `step-6-system` as the explicit anchor).
- All cross-article links use the explicit anchor (`#step-16-kernel-advanced`, `#step-11-finish`, etc.), not the auto-slugged heading.

## Wording (zh-cn ↔ zh-tw)

OpenCC alone is not enough. After conversion, fix these terms by hand:

| zh-cn (mainland) | zh-tw (Taiwan) |
| :--- | :--- |
| 软件 | 軟體 |
| 默认 | 預設 |
| 内存 | 記憶體 |
| 内核 / 内核 | 核心 |
| 数据 | 資料 |
| 网络 | 網路（**not** 網絡）|
| 视频 | 影片 |
| 字体 | 字型 |
| 用户 | 使用者 |
| 服务器 | 伺服器 |
| 文件 | 檔案（context-dependent）|
| 信息 | 訊息 / 資訊 |
| 屏 / 屏幕 | 畫面 / 螢幕 |

Pure OpenCC bugs to watch (we have caught these): `包 → 套件` (wrongly turning `包含` into `套件含`), duplicated trailing characters from word boundaries (e.g. `使用者名稱稱`). The conversion script must skip the inside of `<code>` and Markdown code fences.

## Per-locale defaults

### Mirrors

- **en**: official `https://distfiles.gentoo.org/`, suggest `mirrorselect` for the user's region.
- **zh-cn**: `https://mirrors.bfsu.edu.cn/gentoo/`, mention TUNA and USTC as alternatives. Add `GENTOO_MIRRORS="https://mirrors.bfsu.edu.cn/gentoo/"` to `make.conf` in examples.
- **zh-tw**: `https://ftp.twaren.net/Linux/Gentoo/` (TWAREN).

### Portage tree sync URIs

- **en**: `https://github.com/gentoo-mirror/gentoo.git`
- **zh-cn**: `https://mirrors.bfsu.edu.cn/git/gentoo-portage.git`
- **zh-tw**: `https://ftp.twaren.net/git/Gentoo/gentoo-portage.git`

### Timezone

- **en**: `UTC` (and list `Asia/Tokyo`, `Europe/London`, `Australia/Sydney`, etc., as examples)
- **zh-cn**: `Asia/Shanghai`
- **zh-tw**: `Asia/Taipei`

### Locale / L10N

- **en**: `L10N="en en-US"`, `LINGUAS="en en_US"`
- **zh-cn**: `L10N="en zh zh-CN"`, `LINGUAS="en zh_CN"`
- **zh-tw**: `L10N="en zh zh-CN zh-TW"`, `LINGUAS="en zh_CN zh_TW"`

### Fonts

- **en** ships a broader catalogue: `media-fonts/noto`, `media-fonts/noto-cjk`, `media-fonts/noto-emoji`, `media-fonts/dejavu`, `media-fonts/jetbrains-mono`, `media-fonts/wqy-microhei`.
- **zh-cn / zh-tw** focus on CJK: `media-fonts/noto-cjk`, `media-fonts/noto-emoji`, `media-fonts/wqy-microhei` (and optionally `media-fonts/source-han-sans`).

### Apps shown in Flatpak / desktop examples

- **en**: generic `<app-name>`, `<app-id>` placeholders.
- **zh-cn / zh-tw**: practical examples — `com.qq.QQ`, `com.tencent.WeChat`, `flclash-bin` (Mainland use), QQ/WeChat for both Chinese locales.

### Input methods

- **en**: Fcitx5 + Rime, IBus + Rime; show generic env vars.
- **zh-cn / zh-tw**: same packages but also `app-i18n/fcitx-chinese-addons`, Bopomofo for zh-tw, Pinyin / Cangjie for both.

## Command rules that apply everywhere

These are mistakes we have explicitly caught in past reviews. New content must follow:

- **Stage3 / ISO download** — use the official `latest-stage3-*.txt` index. `wget` does not expand remote wildcards, so any line of the form `wget https://.../stage3-*-*.tar.xz` is **broken** and must be replaced with:
  ```bash
  STAGE3=$(wget -qO- https://distfiles.gentoo.org/releases/<arch>/autobuilds/latest-stage3-<arch>-<profile>.txt | grep -v '^#' | cut -d' ' -f1)
  wget "https://distfiles.gentoo.org/releases/<arch>/autobuilds/${STAGE3}"
  wget "https://distfiles.gentoo.org/releases/<arch>/autobuilds/${STAGE3}.asc"
  gpg --verify stage3-*.tar.xz.asc stage3-*.tar.xz
  ```
- **GPG signing key** — use the current Gentoo Release Engineering Linux Release Signing Key, fingerprint `D99EAC7379A850BCE47DA5F29E6438C817072058`, keyserver `hkps://keys.gentoo.org`. The old short-form `0xBB572E0E2D182910...` and `keys.openpgp.org` references must be removed.
- **LUKS mapper name** — install-base uses `cryptroot` consistently across § 3 (format/mount), § 6.4 (fstab), § 10 (bootloader), § 11 (initramfs). m-series uses `gentoo-root` internally but consistently. Whichever name you pick, **use it everywhere in the same article** — mixing breaks dracut's `rd.luks.name=...=<mapper>` parameter.
- **Partition numbers** — must match the running example throughout the article. m-series caught a real bug where the LUKS partition was sometimes `nvme0n1p5` and sometimes `nvme0n1p6` in the same article; pick one and keep it.
- **`current-stage3-*/` directory does not exist** on `distfiles.gentoo.org`. The correct path is `releases/<arch>/autobuilds/<TIMESTAMP>/...` or `releases/<arch>/autobuilds/latest-stage3-*.txt`.
- **Package atoms (verify on https://packages.gentoo.org/ before adding)**:
  - ccache is `dev-util/ccache`, **not** `dev-build/ccache`.
  - clang / llvm are `llvm-core/clang` and `llvm-core/llvm`; the old `sys-devel/clang` and `sys-devel/llvm` atoms have been moved — do not list both.
  - cmake / meson / ninja moved to `dev-build/`; use `dev-build/ninja`, etc.
  - Fcitx5 is `app-i18n/fcitx` (with `app-i18n/fcitx-rime`, `app-i18n/fcitx-configtool`, `app-i18n/fcitx-chinese-addons`), **not** `app-i18n/fcitx5`.
  - genfstab is `sys-fs/genfstab` (from arch-install-scripts).
  - Display-manager init is `gui-libs/display-manager-init`.
  - zram: OpenRC uses `sys-block/zram-init`; systemd uses `sys-apps/zram-generator`.
- **systemd-boot prerequisites** — `bootctl` needs the `boot` USE flag on `sys-apps/systemd` (systemd ≥ 254) or on `sys-apps/systemd-utils` (`boot kernel-install`) for OpenRC users.
- **`installkernel` USE flag** — `echo 'sys-kernel/installkernel dracut' > /etc/portage/package.use/installkernel` makes dist-kernel rebuilds invoke dracut automatically.
- **`rd.luks.allow-discards`** — only when SSD TRIM through LUKS is acceptable; otherwise omit.

## Content forbidden in every locale

Cleared during the May 2026 review and must not be re-introduced:

- Decorative emoji in section titles, navigation, or prose. Functional UI glyphs like `☰` are fine; everything else is out.
- Inline `<div style="background: linear-gradient(...)">` decoration — the about / timeline pages stay plain Markdown.
- "Community build" image distribution paragraph that pointed at a Google Drive folder.
- "Business" or "商科" anywhere — Zakk's major is `Art` / `艺术` / `藝術`.
- Framing the relationship as "long-distance" / `异地` / `遠距`. Paper is in Australia now; the historical fact that the relationship started long-distance can be mentioned as a milestone but the current state is "together in Australia".

## Localisation conversion checklist

When porting an updated `zh-cn` post to `zh-tw` and then to `en`:

1. **Re-sync `zh-cn`** from the latest source you trust. Apply the link / mirror / app substitutions for Mainland audiences. Run a build and verify all anchors resolve.
2. **Generate `zh-tw`** with OpenCC, then:
   - Replace `BFSU` / `mirrors.bfsu.edu.cn` / `Asia/Shanghai` with TWAREN / `Asia/Taipei`.
   - Apply the wording table above (`內核` → `核心`, `网络` → `網路`, etc.).
   - Hand-scan for the OpenCC `包 → 套件` and `稱稱`/`X X` duplication bugs.
3. **Adapt `en`** from `zh-tw`:
   - Replace Mainland/Taiwan mirrors with official Gentoo distfiles + `mirrorselect`.
   - Replace `Asia/Taipei` with `UTC` and the example list (`UTC, America/New_York, Europe/London, Asia/Tokyo, Australia/Sydney`).
   - Replace `com.qq.QQ` / `com.tencent.WeChat` / `flclash-bin` with generic `<app-name>` placeholders.
   - Drop Telegram/QQ/WeChat community links; link to Gentoo Forums and `#gentoo @ Libera.Chat`.
   - Expand the font set to the international list (Noto, DejaVu, JetBrains Mono).
4. **Verify cross-article anchors** — every internal `/posts/.../#anchor` reference must resolve in the rendered HTML for the same locale. A clean `hugo --minify` rebuild plus a grep for missing `id=...` lines is the standard final check.
5. **Run the bash extractor** (`/tmp/extract_bash.py` style: extract only `bash`/`sh` blocks, strip comments) and `diff` the three locales. The only differences that should remain are:
   - Mirror URLs (BFSU / TWAREN / official)
   - Timezone (`Asia/Shanghai` / `Asia/Taipei` / `UTC`)
   - L10N / LINGUAS values
   - Username placeholders (`zakk` / `<用户名称>` / `<使用者名稱>` / `username`)
   - Mainland / Taiwan / international app examples
   - Optional Mainland-only sections (gentoo-zh overlay, `flclash-bin` demo, app-i18n/fcitx-chinese-addons)

   Anything else in the diff is a bug.

## Verified workflow snippets

These are the canonical versions of common steps; copy from here when in doubt.

**Stage3 (AMD64, OpenRC):**
```bash
STAGE3=$(wget -qO- https://distfiles.gentoo.org/releases/amd64/autobuilds/latest-stage3-amd64-openrc.txt | grep -v '^#' | cut -d' ' -f1)
wget "https://distfiles.gentoo.org/releases/amd64/autobuilds/${STAGE3}"
wget "https://distfiles.gentoo.org/releases/amd64/autobuilds/${STAGE3}.asc"
gpg --keyserver hkps://keys.gentoo.org --recv-keys D99EAC7379A850BCE47DA5F29E6438C817072058
gpg --verify stage3-*.tar.xz.asc stage3-*.tar.xz
tar xpvf stage3-*.tar.xz --xattrs-include='*.*' --numeric-owner
```

**Stage3 (ARM64, systemd, Asahi):** swap `amd64` for `arm64`, `openrc` for `desktop-systemd`.

**chroot prep (install-base / AMD64):**
```bash
cp --dereference /etc/resolv.conf /mnt/gentoo/etc/
mount --types proc /proc /mnt/gentoo/proc
mount --rbind /sys  /mnt/gentoo/sys  && mount --make-rslave /mnt/gentoo/sys
mount --rbind /dev  /mnt/gentoo/dev  && mount --make-rslave /mnt/gentoo/dev
mount --rbind /run  /mnt/gentoo/run  && mount --make-rslave /mnt/gentoo/run
chroot /mnt/gentoo /bin/bash
source /etc/profile
export PS1="(chroot) ${PS1}"
```

**LUKS on btrfs (install-base, mapper name `cryptroot`):**
```bash
cryptsetup luksFormat --type luks2 --pbkdf argon2id --hash sha512 --key-size 512 /dev/nvme0n1p3
cryptsetup luksOpen /dev/nvme0n1p3 cryptroot
mkfs.btrfs --label root /dev/mapper/cryptroot
mount /dev/mapper/cryptroot /mnt/gentoo
```

**dracut for LUKS + btrfs (install-base § 11):**
```ini
# /etc/dracut.conf.d/luks.conf
add_dracutmodules+=" btrfs systemd crypt dm "
```
```bash
dracut --kver "$(make -C /usr/src/linux -s kernelrelease)" --force
```

**Kernel cmdline for LUKS root via systemd-boot:**
```text
rd.luks.name=<LUKS-UUID>=cryptroot root=/dev/mapper/cryptroot rootfstype=btrfs rd.luks.allow-discards init=/lib/systemd/systemd
```

**GRUB cmdline for the same (install-base § 10.1):**
```bash
GRUB_CMDLINE_LINUX="rd.luks.uuid=<LUKS-UUID> rd.luks.allow-discards root=UUID=<decrypted-fs-UUID> rootfstype=btrfs"
```
After modifying:
```bash
grub-mkconfig -o /boot/grub/grub.cfg
```
